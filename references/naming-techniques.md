Riffing, transformation, ownability, and advanced generation techniques. Load this file during Loop 2c (Creative Riffing), Loop 3 (Shortlisting), and Loop 4b (Pool Recovery).

### Key Rules
1. **Apply riffing techniques to DIRECTIONS (themed groups), not individual names** — "the measurement direction" covers Assay + Gauge + Caliber together
2. **Label which technique produced each candidate** — the user should see WHY a name was generated
3. **For every found word, generate 3-5 compound alternatives** — compounds survive validation; found words usually don't

## Systematic Riffing Axes

When exploring variants of a promising name or concept in 2c (Creative Riffing), use these eight transformation axes. Each axis produces a different type of candidate.

| Riff Axis | What You Do | Example (from "navigate") |
|---|---|---|
| **Root compression** | Shorten to a tighter stem | navigate → navi, nav, navan |
| **Etymology** | Use historical/Latin/Greek root | navigate → navis, vela, vector |
| **Metaphor shift** | Move to adjacent symbolic object | navigate → helm, keel, bearing, beacon |
| **Outcome shift** | Name the result, not the act | navigate → arrival, waypoint, northstar |
| **Sensory shift** | Move from one sense to another | navigate → signal, sightline, vantage |
| **Sound tuning** | Make it sharper/softer/faster/heavier | navis → navix, navora, navan |
| **Language shift** | Translate or borrow related roots | navigate → azimuth, meridian |
| **Structural shift** | Change word class or format | navigate → Waypoint, TrueNorth, Helm |

### The Riff Matrix
For each promising direction, systematically ask:

| Start With | Ask | Output Type |
|---|---|---|
| Root | What is the smallest memorable stem? | Compact coined forms |
| Meaning | What object symbolizes this idea? | Metaphor names |
| Benefit | What does the user feel/get? | Outcome names |
| Context | What is the world around the product? | Atmosphere names |
| Language | Related roots in Latin/Greek/Romance/Germanic? | Elevated alternatives |
| Sound | Should this feel sharp, soft, bold, warm, fast? | Phonetic tuning |
| Structure | One word, compound, phrase, clipped form? | Format diversity |

### Riffing Mistakes to Avoid
- **Only** changing prefixes/suffixes (use them as ONE axis among many, not the only move)
- Staying too close to one stem
- Making all variants the same sound shape
- Forcing foreign-language words without real cultural understanding
- Generating 30 names with one exact emotional note

### Worked Example: Riffing from `forge`

This demonstrates the rhythm of a full riffing session — how each axis produces a different flavor, and how the pool diversifies as you go. The seed is a brand brief for a developer tools company. **Do not anchor on these specific outputs.** The point is the progression pattern, not the vocabulary.

**Step 1 — Root compression** (shorten to tighter stems)
- Forg, Forj, Forgn, Forga, Foryx

What changed: shorter, coined, more ownable. "Forj" swaps the soft G for a harder J — shifts from craft to impact.

**Step 2 — Etymology** (Latin/Greek/Germanic roots for "forge")
- Fabrica (Latin: workshop), Vulcan (Roman god of the forge), Ferrum (Latin: iron), Smið (Old Norse: smith), Fucina (Italian: forge)

What changed: moved from English metaphor to classical authority. "Fabrica" and "Fucina" feel European/premium. "Ferrum" feels scientific. Different emotional registers from the same core concept.

**Step 3 — Metaphor shift** (adjacent objects/concepts to forging)
- Anvil, Crucible, Ingot, Bellows, Temper, Alloy

What changed: shifted from the act of forging to the tools and materials around it. "Crucible" carries pressure/transformation. "Temper" carries both heat-treatment and disposition. "Alloy" carries combination/strength.

**Step 4 — Outcome shift** (name the result, not the act)
- Harden, Weld, Cast, Yield, Proof

What changed: shifted from noun-world to verb-world. These are what forging *produces*. "Proof" works on multiple levels (proof of work, bulletproof, proofing metal). "Yield" carries both output and financial return.

**Step 5 — Sound tuning** (vary phonetic texture on the strongest candidates)
- Crucible → Crusible, Krucis, Cruxible
- Temper → Tempra, Tempero, Timpra
- Alloy → Alloix, Aloya, Loy

What changed: created ownable variants by adjusting vowels, swapping consonants, truncating. "Krucis" (K-for-C, Latin inflection) feels sharper and more technical than "Crucible." "Tempra" is cleaner than "Temper" and avoids the common-word problem.

**Step 6 — Coined hybrids** (blend promising roots with other concepts)
- Forgecraft, Ironcast, Cruciform, Alloyant, Tempervault
- Ferrix, Vulcana, Smythic, Forjent, Crucix

What changed: combined the strongest roots with affixes and blends. Some are compounds (Forgecraft), some are pseudo-classical (Ferrix, Crucix), some are blend-derived (Alloyant = alloy + elegant/ant suffix). This is where the most ownable, trademark-friendly candidates emerge.

**Running total: ~35 candidates across 6 axes.** The pool spans found words (Anvil, Temper), classical lifts (Fabrica, Ferrum), coined forms (Tempra, Forjent), and compounds (Forgecraft, Ironcast). No two candidates sound alike. This is the goal of systematic riffing.

### Creative Respelling Patterns
When a conventional spelling is taken, these respelling patterns create ownable variants while preserving pronunciation:
- **K for C**: Klear, Kore, Kraft → harder, more technical feel
- **X for -cks/-cts**: Lynx, Onyx, Nyx → sharper, more modern
- **Y for I**: Lyft, Gyro, Byte → softer, more distinctive
- **Dropped vowels**: Flickr, Tumblr, Scribd → compressed, digital-native
- **Ph for F**: Pharos, Phaze, Phenix → classical, elevated
- **J for G (soft)**: Jiobit (from Geo-), Jenesis → playful, unexpected
- **-ik for -ic**: Optik, Metrik, Logik → Germanic precision, ownable
- **Double letters**: Dribbble, Fiverr → visual distinctiveness, "more features" perception

## Industry-Native Compound Patterns (Tech/SaaS Example)

**The technique is universal; the vocabulary below is tech/SaaS-specific.** Every industry has its own compound vocabulary — the structural patterns that feel credible and natural alongside existing brands in that vertical. The process is always the same:

1. **Study 10+ successful brands in the target vertical.** What compound structures do they use?
2. **Extract 8-12 industry-native compound elements** — suffixes, prefixes, and structural patterns that would feel at home next to the competitive set.
3. **Systematically cross each shortlisted root** with these elements. Not every combination works — scan for natural-sounding ones and discard awkward ones.
4. **Also generate 4-6 cross-industry elements** — the most distinctive compounds borrow from OUTSIDE the industry. "Protocol" for a fashion brand, "Codex" for a dev tool.

**Industry compound examples (generate fresh for each brief — don't reuse these):**

| Industry | Natural compound elements | Example outputs |
|---|---|---|
| **Tech/SaaS** | -works, -craft, -wise, -OS, -ware, -kit, -base, -lab, -stack, -grid | Siftworks, DeepKit, SignalOS |
| **Fashion/lifestyle** | Studio, House, Atelier, & Co, Supply, Goods, Society, Edit | Crow Studio, Moth & Co |
| **Finance/VC** | Capital, Partners, Ventures, Group, Advisory, Holdings | Benchmark Capital |
| **Healthcare** | Health, Therapeutics, Bio, Sciences, Medical, Care, Rx | Lumen Therapeutics |
| **Food/beverage** | Co, Kitchen, Provisions, Pantry, Brewing, Goods | Sloe Provisions |
| **Legal/GRC** | Logic, Systems, Analytics, Intelligence, Protocol | LogicGate, Relativity |
| **Real estate/proptech** | Properties, Spaces, Dwellings, Built, Grid, Field | Fieldwire |
| **Education** | Academy, Institute, Learning, Prep, Labs | Lambda School |

The tables below show the tech/SaaS compound vocabulary in detail — use them when the brief IS tech/SaaS. For all other industries, generate equivalent tables dynamically using the 4-step process above.

Common tech/SaaS naming patterns that consistently produce ownable compound names. Compounds built from these suffixes are far more likely to clear competitive screening than single found words.

**For tech briefs, use these as a parallel generation track alongside found-word generation.** For every promising found word, immediately generate 3-5 compound variants using these patterns.

### System/Infrastructure Suffixes
Names that position the product as a system, engine, or operational layer.

| Suffix | Semantic contribution | Examples | Best for |
|---|---|---|---|
| **-works** | Factory, system, engine | Signalworks, Siftworks, Clockworks | Platforms that process/transform. Industrial authority. |
| **-craft** | Skill, mastery, artisanal quality | Sightcraft, Dealcraft, Minecraft | Products built with intentionality and expertise. |
| **-wise** | Intelligence, direction, knowledge | Siftwise, Bitwise, Streetwise, Clockwise | Products that are smart about something. Systems-thinking energy. |
| **-base** | Foundation, storage, platform | Coinbase, Firebase, Airtable (conceptually) | Data platforms, foundational infrastructure. |
| **-OS** | Operating system, foundational layer | SiftOS, FluxOS (note: check for conflicts — -OS is popular) | Products positioning as "the operating system for X." Directly taps the "infrastructure" metaphor. |

| Suffix | Semantic contribution | Examples | Best for |
|---|---|---|---|

### Domain Hacks
Using the TLD as part of the name. Two distinct patterns:

#### Pattern 1: TLD as suffix
The TLD replaces the last syllable or letters of the word. The domain *is* the name.

| TLD | Example | Word formed | Notes |
|---|---|---|---|
| **.ly** | bit.ly, vend.ly, visual.ly | Bitly, Vendly, Visually | Most productive ccTLD hack. Works with verbs, adjectives, adverbs. |
| **.io** | stud.io, pull.io, portfol.io | Studio, Pullio, Portfolio | Natural when word ends in -io. Developer-friendly. |
| **.ai** | essai → ess.ai, verific.ai | Essai, Verificai | Works when name naturally splits at the TLD boundary. |
| **.co** | angel.co, dri.co | Angel, Drico | .co as shorthand for "company" or word completion. |
| **.is** | th.is, what.is, generat.is | This, Whatis, Generatis | Icelandic ccTLD. Clean for philosophical/identity names. |
| **.me** | about.me, na.me, resu.me | Aboutme, Name, Resume | Personal brands, consumer products, identity. |
| **.it** | do.it, dev.it, sp.it | Doit, Devit, Spit | Italian ccTLD. Works with short action words. |
| **.us** | foc.us, geni.us, stat.us | Focus, Genius, Status | Strong with Latin-root words ending in -us. |
| **.so** | al.so, ver.so, cur.so | Also, Verso, Curso | Less common but distinctive. |
| **.er** | mak.er, build.er, discov.er | Maker, Builder, Discover | Works with agent nouns. Limited registrar support. |
| **.al** | optim.al, minim.al, sign.al | Optimal, Minimal, Signal | Works with -al adjectives and Latin roots. |
| **.to** | pho.to, cryp.to, poin.to | Photo, Crypto, Pointo | Tongan ccTLD. Natural with -to endings. |

#### Pattern 2: TLD as semantic layer
The TLD adds meaning but the name works without it too.

| TLD | Example | Notes |
|---|---|---|
| **.ai** | sentry.ai, proof.ai | Signals AI/intelligence — becoming a de facto tech TLD |
| **.io** | tray.io, socket.io | Signals developer/infrastructure tooling |
| **.co** | stripe.co, hover.co | Signals modern startup — perceived as .com alternative |

#### Generation technique: ccTLD-first naming
When generating names, work backwards from productive ccTLDs:

1. **Pick a target ccTLD** (e.g., `.ly`)
2. **List words that end in those letters** — friendly, deeply, supply, apply, rally, tally, deftly, swiftly
3. **Split at the TLD boundary** — friend.ly, deep.ly, supp.ly, app.ly, ral.ly, tal.ly, deft.ly, swift.ly
4. **Evaluate the prefix as a standalone brand element** — does "friend," "deep," "supp," "ral" carry meaning on its own?
5. **Best candidates** have a prefix that works as a word or suggestive fragment AND a natural pronunciation across the dot

**When this works best:**
- `.ly` — the most versatile; turns almost any verb/adjective into an adverb-brand (clever.ly, exact.ly, rapid.ly)
- `.is` — existential/identity brands (gen.is, what.is, th.is)
- `.us` — Latin-root words with built-in gravitas (foc.us, rad.us, nex.us)
- `.io` — words with Italian/Latin -io endings (stud.io, rat.io, portfol.io)

**Cautions:**
- Domain hacks can cause confusion in verbal communication ("Is it V-E-N-D dot L-Y or V-E-N-D-L-Y dot com?"). Best when the hack is the *primary* domain, not an alternative.
- Some ccTLDs have registration restrictions or political risks (e.g., `.ly` is Libya, `.io` is British Indian Ocean Territory — both have had stability concerns). Research the registry before committing.
- The more natural the split, the stronger the hack. `bit.ly` works because "bit" is a real word. `visu.al` works because the split is clean. `bra.nd` doesn't work because "bra" derails the reading.
- Verbal test: say "check out [name]" aloud. If the listener wouldn't know how to type it, the hack is too clever.

### Y-for-I Respelling (the Lyft/Byte pattern)
The single most productive respelling for tech brands. Replace "i" with "y" to create distinctiveness while preserving pronunciation:
- Sift → **Syft** (note: check Syft alone — often crowded. Compound forms like Syftworks, Syftware are safer)
- Bit → **Byt**
- Sight → **Syght** (harder to read — use sparingly on longer words)
- Lift → **Lyft** ⚠️

One respelling is the right amount. Two (e.g., "Syftwyre") looks try-hard. The Y creates visual distinctiveness and ownability without sacrificing pronounceability.

## Prefix × Suffix Matrix (Systematic Combinatorial Generation)

A formalized methodology for producing ownable compound names by systematically crossing promising prefixes with productive suffixes. **The matrix structure is universal; the elements you put into it must be generated dynamically from each brief.** The tech/SaaS example below shows the process — for other industries, replace both the prefix column and suffix row with elements appropriate to that brief's domain.

**The process:**
1. From the user's preferences, identify 5-8 productive prefixes (e.g., True-, Clear-, Deep-, Iron-, Signal-, Dark-)
2. From the suffix libraries (Productive Affixes + Tech Branding Patterns), identify 8-12 productive suffixes (e.g., -works, -craft, -wise, -mark, -forge, -scope, -vault, -ware, -OS, -base)
3. Cross every prefix with every suffix in a matrix
4. Scan the matrix for combinations that sound natural and carry meaning
5. Flag ⚠️ conflicts inline
6. Present the strongest 15-20 to the user

**Example matrix (partial):**

| | -works | -craft | -wise | -mark | -forge | -scope | -ware | -OS |
|---|---|---|---|---|---|---|---|---|
| **True-** | Trueworks | Truecraft | Truewise | Truemark | Trueforge | Truescope | Trueware | TrueOS |
| **Clear-** | Clearworks | Clearcraft | Clearwise | Clearmark | Clearforge | Clearscope | Clearware | ClearOS |
| **Deep-** | Deepworks | Deepcraft | Deepwise | Deepmark | Deepforge | Deepscope | Deepware | DeepOS |
| **Iron-** | Ironworks | Ironcraft | Ironwise | Ironmark | Ironforge | Ironscope | Ironware | IronOS |
| **Signal-** | Signalworks | Signalcraft | Signalwise | Signalmark | Signalforge | Signalscope | Signalware | SignalOS |
| **Sift-** | Siftworks | Siftcraft | Siftwise | Siftmark | Siftforge | Siftscope | Siftware | SiftOS |

Not every cell produces a winner. Scan for natural-sounding combinations (Siftworks, Ironmark, Deepcraft) and discard awkward ones (Clearwise, Signalforge). The matrix ensures you don't miss strong combinations that intuition would overlook.

**Also cross roots with roots:** Take the user's two favorite roots and combine them: SignalSift, SightForge, CraftMark. These compound-compounds can be surprisingly strong.

## AI-Embedded Naming

Names where "AI" naturally appears within the word — not appended as a suffix ("[name]AI") but organically embedded. This is a distinctly 2024-2026 technique as AI becomes a core product identifier.

**Examples:**
- **Essai** — French for "test." The word naturally ends in "ai." essai.ai is recursive perfection.
- **CuriAI** — Curia (governing assembly) + AI embedded at the end.
- **MosAIc** — Mosaic with AI in the middle.
- **AIsle** — Aisle with AI at the start (stretching).

**The technique:**
1. Take your candidate list and scan for names that contain the letter sequence "a-i" in any position
2. Check if the "ai" can be visually or typographically highlighted without distorting pronunciation
3. The best examples are words where "ai" is a natural part of the spelling — not forced insertions

**Anti-pattern:** Don't force "AI" into words where it doesn't belong (e.g., "AIssay," "TrAIage"). The insertion should feel discovered, not constructed. If you have to capitalize AI in the middle of a word to make it visible, it's probably forced.

## Backronym Generation

Start with a word you want to use, then reverse-engineer a phrase that fits the initials. This gives a found word or coined name an additional layer of meaning and institutional legitimacy.

**Examples:**
- **SIFT** → Screening Intelligence for Fund Triage
- **ASSAY** → Automated Screening & Scoring Analytics for Yield
- **ASICS** → Anima Sana In Corpore Sano (healthy soul in a healthy body)
- **CAPTCHA** → Completely Automated Public Turing test to tell Computers and Humans Apart

**The technique:**
1. Take your strongest name candidates
2. For each, try to construct a phrase where the first letters spell the name
3. The phrase should describe what the product actually does
4. The backronym is a BONUS, not the primary name — the name must work without it

**Anti-pattern:** Don't contort language to force a backronym. "Systematic Yield Framework for Triage" = SYFT works. "Super Yielding Fantastic Triage-ware" = desperation. If the backronym requires awkward word choices, skip it. Not every name needs one.

## Alternate Real Spellings

Different from creative respelling (which invents new spellings). These are actual dictionary-valid alternate spellings of existing words — two legitimate ways to spell the same word.

**Examples:**
| Standard | Alternate | Notes |
|---|---|---|
| Gauge | **Gage** | Both are real. Gage is also a type of plum and means a pledge/security. |
| Grey | **Gray** | Regional variation (British vs American) |
| Adviser | **Advisor** | Both accepted |
| Analyse | **Analyze** | Regional variation |
| Judgement | **Judgment** | Both accepted |
| Colour | **Color** | Regional variation |
| Draught | **Draft** | Different spellings, both real |
| Kerb | **Curb** | Regional — Curb is more ownable in US market |
| Plough | **Plow** | Regional |

**Why this matters for naming:** When "Gauge" is taken by 5 companies, "Gage" may be clear — and it's a real word, not a misspelling. It carries the same meaning with a different visual identity. Always check whether your taken found-word favorites have legitimate alternate spellings.

## Domain-First Naming

When all good words seem taken, invert the process: start from what domains ARE available and work backwards to names.

**The technique:**
1. Identify the user's preferred TLDs (.com, .ai, .co, .io — and ccTLDs if open to domain hacks)
2. Use domain registrar search tools to scan for available domains in the target semantic space
3. Work backwards from available domains to names worth building a brand around
4. For ccTLD hacks: list words ending in the TLD letters, split at the boundary, evaluate (see Domain Hacks section above)

**Example process:**
- Search registrar for available 5-6 letter .com domains containing "sift," "vet," "flux," "true"
- Find siftwise.com is available → evaluate "Siftwise" as a brand name
- Find truevet.com is available → evaluate "Truevet" as a brand name
- Try ccTLD-first: words ending in "-ly" → swift.ly, deft.ly, supp.ly → evaluate as brands

**When to use:** Late in the pipeline after validation has killed most candidates. This inverts the normal flow (name → check domain) to (available domain → evaluate name). It's pragmatic rather than creative, but pragmatism wins when creativity keeps hitting walls. The ccTLD variant is especially productive when .com is saturated in the target space.

**Anti-pattern:** Don't settle for a mediocre name just because the domain is available. The domain is a necessary condition, not a sufficient one. A great name with a slightly imperfect domain (name.ai instead of name.com) beats a forgettable name with a perfect .com.

## The Lexicon Three-Team Method

Inspired by Lexicon Branding's actual naming process. Approach the same brief from three completely different angles simultaneously. This breaks the gravitational pull of "names that sound right for this category" and produces unexpected candidates.

**Team 1 — On-brief:** Generate names directly responding to the brand brief. "What should a VC operations platform be called?" This produces the expected, category-appropriate candidates.

**Team 2 — Competitor:** Generate names as if you're naming a competitor to the brand. "What would Affinity's main rival be called?" This breaks assumptions about what "works" in the category by forcing you to find adjacent-but-different territory.

**Team 3 — Unrelated domain:** Generate names as if the product were in a completely different category. "If this VC platform were a whiskey brand, what would it be called? A watchmaking brand? An observatory? A metallurgy company?" The third team often produces the most unexpected — and ultimately strongest — candidates.

**Why Team 3 works:** It accesses vocabulary and metaphor domains that category-native thinking never reaches. Our test session produced Cupel (metallurgy), Escapement (watchmaking), Proof (whiskey/spirits), and Cask (whiskey) from Team 3 — none of which would have emerged from "name a VC platform."

**Productive Team 3 domains to try:**
- Whiskey/spirits: Proof, Cask, Reserve, Vintage, Barrel, Stillhouse, Dram
- Watchmaking: Escapement, Mainspring, Calibre, Complication, Crown, Bezel
- Observatory/astronomy: Fornax, Transit, Zenith, Aperture, Parallax, Apogee
- Metallurgy: Cupel, Anneal, Temper, Smelt, Crucible, Forge, Assay
- Cartography: Meridian, Azimuth, Isoline, Northing, Benchmark, Survey
- Architecture: Lintel, Quoin, Cupola, Plinth, Spandrel, Keystone
- Maritime: Keelson, Sounder, Leadline, Bollard, Capstan, Ballast

## Morphological Analysis (Zwicky Method)

A systematic combinatorial technique from engineering problem-solving, adapted for naming. Instead of relying on intuition, define the dimensions of the naming problem, list options for each dimension, and systematically cross-combine.

**The process:**
1. Define 4-6 dimensions relevant to the naming brief
2. List 4-8 options per dimension
3. Cross dimensions systematically to generate candidates from every region of the possibility space

**Example dimensions for a VC ops platform:**

| Dimension | Options |
|---|---|
| **Semantic field** | vision, craft, navigation, truth, signal, structure, separation, measurement |
| **Phonetic texture** | hard/decisive, soft/warm, crisp/precise, balanced |
| **Syllable count** | 1, 2, 3 |
| **Word class** | noun, verb, adjective, coined, compound |
| **Language origin** | English, Latin, Greek, French, compound, coined |
| **Naming category** | evocative, found word, compound, metaphorical, coined |

**Crossing:** "2-syllable Latin-origin noun with crisp phonetics from the 'truth' semantic field" → Verity, Verax, Certus. "1-syllable English verb with hard phonetics from the 'separation' field" → Sift, Cull, Sort. This ensures you don't generate everything from the same narrow slice of the possibility space.

**When to use:** After intuitive generation plateaus. The matrix reveals blank spots in your coverage — dimensions you haven't combined yet. It's mechanical but comprehensive.

## Semiotic Territory Mapping

*Confirmed by 3/4 models reviewed (Gemini Thinking, GPT, Codex)*

Map the category across meaning-system tensions BEFORE generating names. Use Greimas-style semiotic squares or simple 2×2 quadrant maps.

**The technique:**
1. Identify 2-3 core tensions in the brand's space (e.g., human vs. machine, speed vs. care, signal vs. noise, institution vs. rebel, precision vs. warmth)
2. Map competitors onto the resulting quadrants
3. Identify the empty quadrant — that's your naming territory
4. Generate names specifically to occupy that quadrant

**Why it matters:** Creates whitespace on purpose instead of discovering it by accident. Prevents "good words that don't differentiate."

**Example tensions for a VC ops platform:**
- Institutional authority ←→ Daily-use approachability
- Mechanical precision ←→ Human judgment
- Active screening ←→ Quiet monitoring

Names generated from the "institutional + active" quadrant feel different from "approachable + quiet" quadrant. The mapping ensures you're choosing, not drifting.

## Classical Stem Engineering

Systematic construction of names from Greek and Latin roots and combining forms. This is distinct from "Foreign-Root" naming (which borrows existing words) — stem engineering BUILDS new words from productive morphemes.

**Productive stems:**

| Stem | Origin | Meaning | Example constructions |
|---|---|---|---|
| **ver-** | Latin (verus) | truth | Veridex, Veritas, Verint, Verax |
| **luc-/lum-** | Latin (lux) | light | Lucerna, Luminary, Lucidia |
| **prae-/pre-** | Latin | before, forward | Praesight, Prevault |
| **vect-** | Latin (vehere) | carry, convey | Vector, Vectis, Vectara |
| **nex-/nect-** | Latin (nectere) | bind, connect | Nexus, Nexora, Nexpoint |
| **cur-** | Latin (cura) | care, attention | Curia, Curator, Curata |
| **spec-/scop-** | Latin/Greek | see, observe | Spectra, Scopus, Prospex |
| **clar-** | Latin (clarus) | clear, bright | Clarion, Clavis, Clarita |
| **fort-/firm-** | Latin | strong, firm | Fortis, Firmament, Fortex |
| **apt-/capt-** | Latin | fit, seize | Aptiva, Captis |

**The process:**
1. Identify the core concept (e.g., "clear seeing" = clar + spec)
2. Combine productive stems with productive suffixes (-is, -ex, -ia, -um, -ix, -ora, -ion)
3. Test for pronounceability and phonetic feel
4. The best constructions feel like they SHOULD be real words — plausible Latin that happens to not exist

**Why it matters:** This is a major agency technique for premium, ownable, globally legible names. The constructions feel institutional and timeless rather than trendy.

**Cliché guard for legal/financial/healthcare verticals:** In these verticals, Latin stems are so overused that the technique's default output may trigger exactly the pattern the user is escaping. When the user rejects "Latin clichés," they usually reject **transparent** Latin (Veritas, Lexicon, Juris) — not **naturalized** Latin (curate, precept, rubric). Test whether each stem's English derivative is more familiar than the Latin root. If so, the construction will read as English, not Latin. Focus on stems like "cur-" (→ curate/curator), "clar-" (→ clarion), "cert-" (→ certify) rather than "ver-" (→ Veritas) or "lex-" (→ Lexicon).

## Structural Sound Patterns

### CVCV Symmetry
Consonant-Vowel-Consonant-Vowel structures have the highest cognitive processing fluency across languages and scripts. They feel "native" in almost every language.

**Examples:** Uber, Meta, Data, Zara, Lego, Nike, Visa, Roku, Hulu, Dara, Velo, Moda

**Why it matters:** If you need maximum global pronounceability and memorability, CVCV names are the safest structural bet.

### Tautonyms / Reduplicatives
Names with repeating sounds or syllables. Achieve maximum memorability through rhythmic repetition.

**Examples:** TikTok, Lulu(lemon), BonBon, Murmur, Cancan, Dodo, JioJio

**Why it matters:** Extreme simplicity, high recall, strong in audio-first contexts (podcasts, voice assistants).

### Rhetorical Device Naming
Use literary/rhetorical devices intentionally as naming patterns:

| Device | Definition | Example |
|---|---|---|
| **Alliteration** | Repeated initial consonant | ForgeFlow, SignalSift, ClearCraft |
| **Assonance** | Repeated vowel sounds | TrueBlue, DeepSeed, SightLine |
| **Consonance** | Repeated consonant sounds | Siftcraft, Darkmark, Deepdeck |
| **Internal rhyme** | Rhyme within the name | TrueCrew, SiftSwift |

**Why it matters:** Rhetorical patterning boosts recall, verbal stickiness, and "pitchability" — how well the name works when spoken aloud in a meeting.

## Contrarian / Anti-Category Naming

Name AGAINST category expectations. If every competitor uses polished, abstract tech language, use plainspoken, blunt, even slightly awkward names. Strategic plainness can be more distinctive than cleverness.

**Examples:**
- **Bench** (bookkeeping) — deliberately plain in a sea of "QuickBooks," "FreshBooks," "Xero"
- **Basecamp** (project management) — grounded, physical in a sea of abstract SaaS names
- **Notion** (productivity) — a common word where competitors used coined names
- **Linear** (issue tracking) — mathematical simplicity vs. Jira, Asana, Monday

**The technique:**
1. Map the naming conventions of the competitive set
2. Identify the dominant pattern (abstract? coined? compound? warm?)
3. Generate names that deliberately violate the pattern
4. The "wrong" name for the category often becomes the most memorable name in the category

**When to use:** Crowded markets where "more of the same" naming guarantees invisibility.

## Conceptual Blending (Fauconnier & Turner, expanded)

Blend mental models, not just syllables. The strongest names fuse two previously unrelated conceptual frames — the innovation happens at the idea level, then gets expressed as a word.

**The technique:**
1. Identify two conceptual frames relevant to the product (e.g., "navigation + intelligence," "craft + infrastructure," "medicine + screening")
2. Ask: "What new meaning emerges when these frames collide?"
3. Find the word or compound that captures the emergent meaning

**Examples:**
- **BlackBerry** = black (sleek, premium) + berry (organic, small, natural) → blended: premium but approachable technology
- **Snowflake** = snow (unique, crystalline) + flake (individual unit) → blended: unique data structures
- **Siftworks** = separation (screening) + factory (system) → blended: the industrial system of intelligent filtering

**Different from Lexical Blends:** Lexical blends combine word sounds (Instagram = instant + telegram). Conceptual blends combine MEANINGS, and the name can be any form — compound, found word, coined — as long as it expresses the blended concept.

## Bouba-Kiki Effect (Sound-Meaning Mapping)

*Confirmed by 2 models. Expand the existing Phonesthetic Framework with this specific finding.*

Cross-cultural research (Ramachandran & Hubbard, 2001) shows that people consistently match rounded shapes to "bouba" and angular shapes to "kiki" — even across languages. This extends to brand names:

| Sound type | Shape association | Brand personality | Example names |
|---|---|---|---|
| Rounded vowels (o, u) + soft consonants (m, l, n) | Soft, curved, large | Warm, trustworthy, approachable | Google, Loom, Murmur |
| Sharp vowels (i, e) + hard consonants (k, t, p) | Angular, small, precise | Technical, fast, innovative | Stripe, Kiln, Pixel |
| Mixed | Balanced | Versatile | Siftworks (hard S + soft W) |

**Practical application:** Map your target brand personality to a shape profile, then select phonemes that match. If the brand needs to feel "precise and institutional," favor angular sounds. If it needs to feel "warm and human," favor rounded sounds.

## Brand Architecture & System Name Family Design

*Confirmed by 3 models.*

A name doesn't exist in isolation — it must spawn coherent sub-names for products, features, tiers, and future extensions. Test naming grammar BEFORE committing.

**Architecture types:**

| Type | Structure | Example |
|---|---|---|
| **Monolithic/Masterbrand** | Parent name + descriptor | Google Docs, Google Sheets, Google Drive |
| **Endorsed** | Sub-brand "by" parent | Courtyard by Marriott |
| **House of Brands** | Independent names, parent hidden | Tide, Pampers, Gillette (all P&G) |
| **Hybrid** | Mix based on product tier | Apple (masterbrand for iPhone, iPad) + Beats (acquired, kept separate) |

**The naming grammar test:**
For each finalist name, simulate:
- "[Name] for Deal Screening" — does it work as a product descriptor?
- "[Name] Insights" / "[Name] Analytics" — do feature names feel natural?
- "[Name] Pro" / "[Name] Enterprise" — do tier names sound right?
- "I use [Name]" / "Check [Name]" — does it work in conversation?
- "[Name] Scout" / "[Name] Signal" — can you name AI agents/features coherently?

If the name can't extend cleanly, it will break at product expansion or platformization.

## Morphological Productivity Planning

Evaluate whether a name can spawn intuitive derivatives — verb forms, user nouns, feature families. Critical for product-led growth.

**Tests:**

| Test | Question | Good example | Bad example |
|---|---|---|---|
| **Verb form** | Can users "verb" the name? | "Slack me" / "Google it" | "Palantir me" (no) |
| **User noun** | What are users called? | "Slackers" / "Notion users" | Hard to derive from most coined names |
| **Adjective** | Can you describe something as [name]-like? | "Very Stripe-like" (clean, precise) | Harder with compounds |
| **Feature family** | Do sub-names feel coherent? | GitHub Actions, GitHub Copilot | — |
| **Hashtag** | Does #[name] work? | #Notion, #Figma | #Sightcraft (fine) |
